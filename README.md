# Random Password Generator

**Автор:** [Аделя Рашитова]

## Описание

GUI-приложение для генерации безопасных случайных паролей с возможностью настройки параметров и сохранением истории.

### Основные возможности:
- Генерация паролей с настраиваемыми параметрами
- Выбор типов символов (буквы, цифры, специальные символы)
- Регулировка длины пароля (4-32 символа)
- Сохранение истории в JSON
- Экспорт истории
- Копирование пароля в буфер обмена
- Гарантированное включение всех выбранных типов символов

## Требования

- Python 3.6+
- tkinter (входит в стандартную библиотеку Python)

  ## Примеры использования

### Сценарий 1: Быстрая генерация стандартного пароля
1. Оставьте настройки по умолчанию (длина 12, выбраны все типы символов).
2. Нажмите кнопку **«Сгенерировать пароль»**.
3. Скопируйте пароль в буфер обмена с помощью кнопки 📋.
4. Используйте пароль для регистрации на сайте или смены пароля учётной записи.

### Сценарий 2: Создание пароля только из цифр
1. Снимите галочки с **«Буквы (a-z, A-Z)»** и **«Специальные символы (!@#$%^&*)»**.
2. Установите длину пароля 6 с помощью ползунка.
3. Нажмите **«Сгенерировать пароль»** — получится 6‑значный цифровой код (например, `729415`).
4. Подходит для PIN‑кодов или простых защитных кодов.

### Сценарий 3: Создание сложного пароля для важного аккаунта
1. Установите длину пароля на 20.
2. Убедитесь, что выбраны все три типа символов.
3. Сгенерируйте пароль — он будет содержать буквы разного регистра, цифры и спецсимволы (например, `K7$mP9xQ#vR2wL5@nF8`).
4. Скопируйте и сохраните пароль в менеджере паролей.

### Сценарий 4: Работа с историей паролей
1. Сгенерируйте несколько паролей с разными настройками.
2. Просмотрите их в таблице **«История паролей»** (последние сверху).
3. При необходимости очистите историю кнопкой **«Очистить историю»**.
4. Экспортируйте историю в файл JSON кнопкой **«Экспорт в JSON»** для резервного копирования.

### Юзабилити‑тесты


1. **Интуитивность интерфейса:** все элементы управления (ползунок, чекбоксы, кнопки) имеют понятные подписи на русском языке.
2. **Обратная связь:** каждое действие пользователя сопровождается визуальной или текстовой обратной связью (сообщения об ошибках, успехи, подсказки).
3. **Стабильность:** приложение не зависает при выполнении операций.
4. **Восстановление состояния:** при перезапуске приложения история паролей загружается из файла `password_history.json`.
5. **Удобство копирования:** кнопка копирования (📋) расположена рядом с полем пароля и легко находится взглядом.

### Технические тесты


1. **Сохранение истории:** после закрытия приложения и повторного запуска ранее сгенерированные пароли отображаются в таблице истории.
2. **Целостность данных:** файл `password_history.json` содержит валидный JSON и не повреждается при многократных операциях сохранения.
3. **Обработка ошибок:** при невозможности сохранить историю (например, из‑за отсутствия прав записи) отображается сообщение об ошибке с описанием проблемы.
